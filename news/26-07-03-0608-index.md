# Daily Brief — 26-07-03 รอบ 06:08 (Bangkok)

## Theme ของรอบ

**48 ชั่วโมงล่าสุด (29 มิ.ย. – 1 ก.ค. 2026) = "agent runtime + governance layer กำลังกลายเป็น battleground ใหม่"**. หลังจากรอบก่อนที่ pattern เป็น distribution moat ผ่าน SI firm (Globant, PwC), รอบนี้ pattern shift ไปที่ **layer ที่ agent วิ่งจริง** — runtime, gateway, application layer, deployment scale

**Anthropic ปล่อย 2 dot ในสัปดาห์เดียว** (29-30 มิ.ย.) เป็น signal เด่นสุด — (1) **Claude apps gateway** เป็น self-hosted control plane ที่ enterprise เอา Claude Code ไปรันในบ้านตัวเองได้: stateless container + Postgres, OIDC (Okta/Entra/Google Workspace), spend cap รายวัน/สัปดาห์/เดือน, provider failover Bedrock ↔ GCP ↔ Anthropic API; (2) **Claude Sonnet 5** — most agentic Sonnet ever, 80.4% Terminal-Bench (แซง Opus 4.8), $2/$10 intro pricing (ถูกกว่า Opus 5 เท่า), ตั้งเป็น default ให้ทุก user ตั้งแต่ launch. คือ Anthropic ครบทั้ง 4 layer: frontier model → MCP protocol → SI delivery network → self-hosted enterprise runtime = **vertically integrated Claude platform** ที่ตอบคำถาม CIO ครบตั้งแต่ "โมเดล? extensibility? deploy? governance?"

**Oracle Fusion SCM Agentic Applications** (29 มิ.ย.) เป็น proof point ของ pattern ใหญ่กว่า — enterprise SaaS ยักษ์ทั้งหมด (SAP Joule Studio, Salesforce Agentforce Studio, ServiceNow Now Assist Skills, Microsoft Agent 365, ตอนนี้ Oracle AI Agent Studio) กำลัง **package agent เป็น "application" ไม่ใช่ "feature"**. 4 apps ใหม่ครอบ Inventory, Supplier Qualification, Production Readiness, Kanban Admin — position ที่ SAP dominate มา 30 ปี. Oracle เลือกเดินสาย SCM ก่อน Finance/HR เพราะ **SCM คือ layer ที่ agent value visible สุด** (stockout ↓, cycle time ↓, service level ↑ — KPI ที่ CFO แคร์ตรง ๆ)

**Extreme Networks Agent ONE Coworker rollout 1 ก.ค.** ให้ทุกลูกค้า Platform ONE พร้อมกันโดยไม่ต้อง opt-in — first at-scale rollout ของ agent สาย infrastructure (network operations). สิ่งที่ Extreme วางไว้ต่างจาก AIOps เดิมคือ **"Nudge"** — agent เสนอ action ก่อน user ถาม (detect Wi-Fi congestion → เสนอ config change; POS slowdown → เสนอ QoS rule). Signal: **infrastructure agent (NetOps, DevOps, SRE) กลับกลายเป็น category ที่ deploy at scale ก่อน knowledge agent** เพราะ telemetry structured + action well-defined + risk observable

Pattern หลักของ 4 ข่าวรวมกัน — **"agent-as-application ที่รันในบ้านลูกค้า" กลายเป็น default deployment model**. Anthropic ให้ enterprise host gateway เอง. Oracle package agent เป็น app ใน Fusion Cloud. Extreme bake agent เข้า console เดิมของลูกค้า. ไม่มีใคร pitch "agent เป็น service ที่คุณ subscribe แล้ว call API ไป endpoint ของ vendor" อีกแล้ว — เพราะ (1) CIO ไม่อยาก manage vendor login/policy/data mapping เพิ่ม, (2) governance ต้อง enforce ใน perimeter ของบริษัท, (3) integration friction ของ standalone agent สูงเกินไป

## TL;DR

- **Anthropic Claude apps gateway = layer ที่ Anthropic แข่ง OpenAI/Google ที่ enterprise governance** — layer นี้ OpenAI ยัง require ให้ผ่าน ChatGPT Enterprise console บน cloud ของ OpenAI, Google ต้องผ่าน Google Cloud console เท่านั้น. Anthropic ปล่อย first-party self-hosted control plane (ไม่ใช่แค่ SDK) = pattern vertically integrated stack ที่ CIO ซื้อครบในที่เดียว. **Builders** ต้อง support OIDC token + route ผ่าน internal gateway + respect spend cap policy — hard-code endpoint ไป Anthropic API/Bedrock ตรง ๆ ใช้ใน Fortune 500 ไม่ได้แล้ว. **Business** ในไทย (ธนาคาร ประกัน โรงพยาบาล) ที่ pilot Claude Code — download binary, config OIDC, ต่อ Bedrock ที่ Singapore region, spend cap ช่วยตัด surprise bill ทันที

- **Sonnet 5 คือช่วงที่ "agent-grade intelligence" กลายเป็น commodity ราคา mid-tier** — SWE-bench Pro 63.2%, BrowseComp 84.7%, Terminal-Bench 80.4% (แซง Opus 4.8), ราคา $2/$10 intro. Pattern ที่เห็น: model tier compression — ช่องว่าง frontier ↔ mid-tier แคบลงเรื่อย ๆ, premium tier กำลังกลายเป็น "edge case optimizer" ไม่ใช่ default choice. **Framework builders** ที่ hard-code Opus/GPT-4o ควรตั้ง Sonnet 5 เป็น recommended default + Opus เป็น escalation. **Business ในไทย** ที่ค้าง pilot เพราะ TCO ไม่ผ่าน — วันนี้เลขเปิดกลับมาน่ารันใหม่. เตือน: intro pricing $2/$10 หายหลัง 31 ส.ค. — commit long-term ต้อง rate lock

- **Oracle Fusion SCM = enterprise SaaS ยักษ์ package agent เป็น "application" ไม่ใช่ "feature"** — pattern เดียวกับ SAP Joule Studio, Salesforce Agentforce Studio, ServiceNow Now Assist Skills, Microsoft Agent 365. Playbook ร่วม: (1) เปิด Studio ให้ customer build agent เอง, (2) ปล่อย pre-built agent เป็นตัวอย่าง, (3) partner ecosystem ต่อผ่าน MCP. **Vertical agent builders** ใน SCM (Fero Labs, Osmos, Kinaxis Aera, o9) ต้อง prove differentiator หรือ pivot เป็น app-on-Studio. **ธุรกิจไทยที่รัน Oracle Fusion** (PTT, SCG, CPF, TOA) — คุย Oracle Country Manager ให้ demo Inventory Command Center + Supplier Qualification บน data ตัวเอง; time-to-value 3-6 เดือน (แทน 12-18 เดือน greenfield)

- **Extreme Agent ONE Coworker rollout = infrastructure agent เป็น category ที่ deploy at scale ก่อน knowledge agent** — telemetry structured อยู่แล้ว (SNMP, syslog, NetFlow), action well-defined (VLAN, QoS, reboot), risk observable ทันที. Pattern ทั้งตลาด — Cisco Splunk Assistant, Juniper Marvis 2.0, HPE Aruba GreenLake Assistant, Extreme Agent ONE — pre-installed with existing platform, integrate กับ management console, target NetOps engineer shortage (~300K job vacancy). **Standalone agent framework** สำหรับ NetOps/DevOps/SRE ต้อง OEM ผ่าน network vendor หรือขายผ่าน hyperscaler marketplace — direct developer signup ตายเพราะ Fortune 500 ห้าม third-party agent มี network access

- **Pattern หลักของ 4 ข่าวรวมกัน — "agent-as-application ที่รันในบ้านลูกค้า" กลายเป็น default deployment model** — Anthropic ให้ host gateway เอง. Oracle package เป็น app ใน Fusion Cloud. Extreme bake เข้า console เดิม. ไม่มีใคร pitch "agent = SaaS ที่คุณ call API ไปที่ vendor" แล้ว — เพราะ CIO ไม่อยาก manage vendor login/policy/data mapping เพิ่ม, governance ต้อง enforce ใน perimeter ของบริษัท, integration friction ของ standalone agent สูงเกินไป. **Signal ต่อ platform players** — control plane อยู่ที่ไหน = distribution moat ยาวสุด, model quality เป็น cost-of-entry

- Action ตรงสำหรับ **AI Agent Platform builders**:
  - (1) **Self-hosted/hybrid deployment** เป็น first-class — ปล่อย binary/container ที่ enterprise รันในบ้านตัวเอง + integrate OIDC/RBAC/spend cap; SaaS-only ตายในตลาด Fortune 500
  - (2) **Re-benchmark ทุก 90 วัน** — model tier compression เร็ว, ราคาถูกลง 3-5 เท่าใน quarter เดียว, default model choice ของ framework ต้อง update อย่างน้อยรายไตรมาส
  - (3) **App-on-Studio strategy** — ถ้าเป็น vertical agent (SCM, HR, Finance, ITSM), เลือก platform ecosystem 1-2 ตัวเพื่อ integrate ลึก (Oracle AI Agent Studio, Salesforce Agentforce, ServiceNow Skills, SAP Joule Studio); standalone ยากจะเจาะ install base
  - (4) **Infrastructure agent zone** = distribution ผ่าน network/cloud vendor เท่านั้น — direct developer signup ตาย, OEM/marketplace only

- Action ตรงสำหรับ **Business / users** ที่ deploy AI Agent:
  - (1) **Insist self-hosted option** ใน RFP — vendor ที่ให้แค่ SaaS API endpoint = red flag governance, spend cap, audit
  - (2) **Cross-check MCP-native** — ทุก vendor SaaS ยักษ์ (Salesforce, ServiceNow, Oracle, SAP, Microsoft) ตอนนี้ open MCP; agent framework ที่ยัง require custom SDK จะเสีย deal
  - (3) **Re-price agent business case ทุก quarter** — model cost ลง 50-70% ต่อปี, TCO ที่คำนวณเมื่อ 6 เดือนก่อนไม่ตรงแล้ว; POC ที่เคยไม่ผ่านให้ re-run
  - (4) **Infrastructure agent (NetOps, DevOps, SRE) = pilot ง่ายสุด** — telemetry มีอยู่แล้ว, ROI วัดจาก mean-time-to-resolution ทันที; category นี้เริ่มได้เร็วกว่า knowledge agent
