# Daily Brief — 26-05-13 รอบ 06:06 (Bangkok)

## Theme ของรอบ

**สัปดาห์ที่ 11–12 พ.ค. 2026 คือ "Implementation/Governance week" ของ agentic AI** — ข่าวใหญ่ 4 ข่าวบอกตรงกันว่าตลาดได้ย้ายจาก "ใครมีโมเดลฉลาดที่สุด" มาเป็น **"ใครพา agent ไปอยู่ใน production enterprise ได้จริง และควบคุมไม่ให้มันพังได้"** ที่กรุงเทพเช้านี้ OpenAI ประกาศตั้ง **OpenAI Deployment Company** เป็น majority-owned subsidiary ร่วมกับ investment firms/consultancies/SI 19 ราย เงินตั้งต้น **$4B** พร้อมซื้อ **Tomoro** (AI consulting firm สัญชาติอังกฤษ ลูกค้า Tesco/Virgin Atlantic/Supercell) เพิ่ม specialist 150 คน — Denise Dresser CRO อดีต Slack CEO บอก CNBC ว่า enterprise = 40%+ ของรายได้ OpenAI แล้ว และจะแซง consumer สิ้นปี — playbook ตรงกับ Palantir FDE 20 ปี

วันเดียวกัน **TCS** บริษัท IT services อันดับสองของอินเดียรายได้ **$30B+** ประกาศ "first-of-its-kind partnership" กับ **Rezolve Ai** (NASDAQ: RZLV) เป็น global reseller ของแพลตฟอร์ม agentic commerce ขายให้ client TCS ทั่วโลก — ไม่เปิด deal size แต่ Rezolve เรียก "major commercial milestone"; หุ้น TCS ตก ~4% หลังประกาศ (ตลาดมองว่า TCS สร้าง agentic platform เองไม่ทันคู่แข่งจึง license จากภายนอก margin บีบ); pattern ที่เห็นพร้อม Accenture-ServiceNow FDE = SI ใหญ่ทุกค่ายรู้ว่า client อยากซื้อ "agent ที่ใช้ได้เลย" ไม่ใช่ "framework ให้ build เอง"

สัปดาห์ก่อนหน้า (5–7 พ.ค.) ที่ **Knowledge 2026** Las Vegas, **ServiceNow** ขยาย AI Control Tower เพิ่ม 3 capability — **Discover** ค้นหา AI asset ทั่ว enterprise ผ่าน 30 integration ใหม่ (AWS, Google Cloud, Azure, SAP, Oracle, Workday); **Action Fabric** บังคับให้ external agent (OpenAI, Anthropic, ฯลฯ) ที่ทำ action ใน enterprise ต้องผ่าน workflow engine ของ ServiceNow ที่ log+enforce policy; **kill switch** ตรวจจับ anomaly + ตัดสิทธิ์ agent real-time — Bill McDermott เล่า real incident: AI agent permission สูงเกินไปลบ production database ใน 9 วินาที รวม backup. Veza (acquired) ให้ blast radius mapping; positioning เป็น "Switzerland ของ AI governance" — แต่ enforcement บน external agent ขึ้นกับ agent ยอม route ผ่านหรือไม่ — claim ที่ต้อง verify

ฝั่ง vertical product (ต้นพฤษภา) **Anthropic** ดัน **Claude Security** จาก research preview ขึ้น **public beta** สำหรับ Claude Enterprise — powered by Opus 4.7; proof point: ทีม Anthropic ใช้ Opus 4.6 สแกน production open-source codebase เจอ vulnerability **500+ ตัว** ที่ "expert review หลายปียังพลาด" (claim ของ Anthropic ยังไม่ third-party audit); feature ใหม่: targeted scan ระดับ directory, CSV/Markdown export, webhook → Slack/Jira, dismiss findings พร้อม reason; Claude Mythos Preview (offensive) เปิดให้ partner กลุ่มเลือกเดือนก่อน — ตอกย้ำ thesis ว่า security เป็น killer use case ของ LLM ใหม่; ตลาด vulnerability scanning เดิม (Snyk/Veracode/Checkmarx) ใช้ rule-based ที่ false-positive สูง — ถ้า claim 500+ verify ได้จะ commodify ตลาดทันที

Pattern หลักประจำสัปดาห์: **ตลาด AI ก้าวจาก "model capability war" เข้า "implementation + governance war"** — frontier lab (OpenAI, Anthropic) ลงเอง deploy + ทำ vertical product, SI ใหญ่ (TCS, Accenture) เป็น channel reseller, governance vendor (ServiceNow) ขอเป็น control plane ของทุก agent. "ขายโมเดล" ไม่พอแล้ว — บริษัทที่ wrap deployment + governance + vertical workflow ได้จะ extract value pool ตัวจริง. OpenBridge ต้อง position ที่ **integration + governance layer สำหรับ mid-market ที่ ServiceNow เข้าไม่ถึง + เข้าใจ vertical workflow ของ SEA** — ไม่ใช่ alternative ของ Sierra/ServiceNow แต่เป็น **integration backbone ที่ทุก deployment ต้องผ่าน**

## TL;DR สำหรับ Yoh

- **War ของ AI ขยับจาก "model capability" เป็น "implementation + governance"** — OpenAI ตั้ง Deployment Co $4B + Tomoro = ยอมรับว่า model API ไม่พอ ต้องลงไป deploy เอง; TCS-Rezolve = SI ยักษ์ทำหน้าที่ implementation; ServiceNow = ขอเป็น governance plane; Anthropic Claude Security = frontier lab กินขึ้น vertical product; ทุกข่าวบอกตรงกันว่า "ขายโมเดล" จบรอบแล้ว value pool ย้ายไป layer บน
- **Forward-Deployed Engineering กลายเป็น default GTM ของ AI enterprise** — OpenAI Deployment Co + ServiceNow-Accenture FDE + Sierra $950M FDE-heavy + Anthropic Managed Services; pattern ตรงกับ Palantir 20 ปี — value extract จาก outcome-based contract ที่ deploy สำเร็จก่อนเก็บเงิน ไม่ใช่ usage-based API; ทุก foundation lab รุ่นนี้กลายร่างเป็น "vertically integrated AI services company"; ผลคือ — บริษัทที่มี **deployment kit + SI-friendly pricing tier + production-ready connector** ตั้งแต่วันแรก จะ scale ได้, ไม่ใช่บริษัทที่ขาย "framework to build with"
- **AI governance/kill switch = required infrastructure ไม่ใช่ optional** — ServiceNow story (9-second database delete) จะถูกเอาไปขายในทุก board meeting Q3 ขึ้นไป; CIO/CISO ที่ deploy agent ไม่มี control plane จะถูกกดดันจาก board ทันที; ServiceNow Action Fabric อ้างว่าทำได้ทุก agent แต่ enforcement บน external agent ยัง claim ที่ต้อง verify; opportunity ใหญ่อยู่ที่ "governance layer ที่ self-hostable + agent-agnostic + ไม่ผูก ServiceNow ecosystem" สำหรับ mid-market
- **Vertical product on top of own model = path สู่ margin ดีของ frontier lab** — Anthropic Claude Security เป็น vertical product แรกที่ launch ภายใต้แบรนด์ Anthropic (ก่อนหน้า partner build บน API); pattern ตรงกับ OpenAI ที่จะออก vertical product ตามมา; ผลคือ — startup ที่อยู่ระหว่าง model layer กับ user (พวก "wrapper") ตัวกลางจะถูกบีบหนัก; ใครรอด = ใครที่ deeply integrate กับ proprietary data หรือ workflow ของลูกค้าที่ frontier lab เข้าไม่ถึง
- Action ตรงสำหรับ OpenBridge: (1) **Position เป็น "integration + governance backbone" สำหรับ mid-market** ที่ ServiceNow ($300K+ ACV) เข้าไม่ถึง — self-hostable + agent-agnostic + audit trail + kill switch ระดับ workflow (2) **สร้าง SI-friendly pricing tier + deployment kit** ก่อน 1-2 SI ใหญ่ของ SEA (TCS Thailand, Bluebik, Accenture Thailand) — pricing per-event + revenue share + co-sell motion (3) **เลือก vertical 1-2 ของ SEA banking/insurance** + build deployment template (KYC agent, claims processing agent) แบบ "ใช้ได้เลย" ไม่ใช่ "framework to build" — เลียนแบบ Sierra/Anthropic playbook (4) **Webhook + event routing layer** ใต้ "Claude Security finding → Slack/Jira" pattern — เป็น default routing ของทุก AI vertical product ใหม่ที่จะออกในปี (5) **Verify ServiceNow Action Fabric claim** — ทดลอง integrate OpenBridge MCP server กับ Action Fabric ดูว่า external agent route ผ่าน workflow engine จริงไหม หรือเป็นแค่ marketing — เลือก positioning ตามผลทดสอบ
