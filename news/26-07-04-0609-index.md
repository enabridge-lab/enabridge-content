# Daily Brief — 26-07-04 รอบ 06:09 (Bangkok)

## Theme ของรอบ

**48 ชั่วโมงล่าสุด (29 มิ.ย. – 2 ก.ค. 2026) = "unit economics ของ agent กลายเป็นสมรภูมิที่ทุกชั้นของ stack ต้อง defend"**. หลังจากรอบก่อนที่ pattern เป็น "agent-as-application ที่รันในบ้านลูกค้า" (runtime/gateway/app layer), รอบนี้ pattern shift ไปที่ **เศรษฐศาสตร์ที่ทำให้ deployment เหล่านั้นคุ้ม** — cost per query, pricing model, tooling velocity, security overhead

**Cisco แจก AI agent ให้พนักงาน 90,000 คน** (1–2 ก.ค.) เป็น signal เด่นสุดฝั่ง demand — Fortune 500 rollout ใหญ่ที่สุดครั้งหนึ่งของปี, แต่ CFO Mark Patterson ตั้ง architecture ไว้ที่ **efficient routing** ("we're not going to burn a whole bunch of tokens with frontier models") — routing layer เลือกโมเดลตาม intent, frontier เป็นข้อยกเว้น. ตัวเลข concrete: AI ร่าง MD&A 80–90% ของ first draft, FY26 guidance $9B, หุ้นขึ้น 53% YTD. Cisco กำลังเปิด playbook ให้ Fortune 500 อื่น: **agent scale ไม่ใช่การซื้อ frontier subscription ให้ทุกคน — คือการสร้าง agent runtime + model gateway ในบ้าน**

**Salesforce Agentforce Help Agent เข้า GA เดือนกรกฎาคม 2026 พร้อม pay-per-resolution $2** เป็น pricing shift ที่จะเขย่าทั้งวงการ — คิดเงินเฉพาะเมื่อ agent ปิดเคสเอง ถ้า escalate to human หรือ walk away = ไม่คิดเงิน. Context: Agentforce ทำ ARR $2.9B, ปิด 29,000+ deals — และ pricing shift นี้ transfer risk จาก buyer (ที่กลัวจ่ายค่า pilot ไม่ได้ผล) กลับไปยัง vendor. คู่แข่งจะออก pricing แบบเดียวกันภายใน 2 quarter หรือถูก dismiss ว่า "ยังคิดแบบ per-message อยู่"

**SnapLogic MCP Builder เข้า GA 1 ก.ค.** — auto-generate MCP server จาก integration pipeline เก่า, OpenAPI spec, และ API Management service ที่มีอยู่แล้ว, ไม่ต้อง rewrite ไม่ต้องเขียน code. มาพร้อม Trusted Agent Identity (propagate user permission) + AI Gateway (audit MCP traffic) + 1,000+ enterprise connector. Signal: **iPaaS (SnapLogic, MuleSoft, Boomi, Workato) กำลัง reposition เป็น "agent middleware"** — MCP adoption จะไม่ถูก driven โดย developer สร้าง server ทีละตัวอีกต่อไป, แต่จะถูก driven โดย integration platform ที่ประกาศ "one-click MCP publish"

**Straiker ปิด Series A $64M** (29 มิ.ย. รายงานทางการ) นำโดย Marathon Management Partners + Citi Ventures + Illuminate Financial + Workday Ventures — traction: **15× revenue ในไม่ถึงปี**, ลูกค้า Fortune 500 หลายราย + frontier AI lab. IDC ประเมิน enterprise จะ deploy AI agent เกิน **1 พันล้านตัวภายใน 2029** (40× จาก 2025) — **agent security กลายเป็น category แยก** ไม่ใช่ sub-feature ของ endpoint/DLP/SIEM เดิม เพราะ agent สร้าง blast radius ที่เครื่องมือเดิมมองไม่เห็น

Pattern หลักของ 4 ข่าวรวมกัน — **ทุก layer ของ agent stack กำลังถูกบีบให้ prove unit economics**: model layer (Cisco: routing > frontier), pricing layer (Salesforce: outcome > activity), tooling layer (SnapLogic: velocity > custom build), security layer (Straiker: agent-native > endpoint reuse). ครึ่งแรกของ 2026 ตลาดพิสูจน์แล้วว่า "agent ทำงานได้"; ครึ่งหลังจะพิสูจน์ว่า "agent คุ้มเงินได้อย่าง scale"

## TL;DR

- **Cisco 90,000 rollout = playbook ที่ Fortune 500 ทั่วโลกจะยกมาอ้าง** — "personal agent for everyone" ทำได้เมื่อคุณสร้าง routing layer ใน middle ไม่ใช่ยิง frontier ทุก query. **Builders**: routing layer + model gateway (Portkey, LiteLLM, Databricks AI Gateway) จะเป็น commodity บังคับใน enterprise stack; framework ที่ hard-code Anthropic/OpenAI endpoint จะโดน bypass. **Business ในไทย** ที่กำลัง pilot ChatGPT Enterprise / Claude Enterprise — ก่อน commit sub, ประเมินก่อนว่า workload 80% เป็น routine ที่ Haiku/Luna/Terra ทำได้ไหม, budget ต่างสัปดาห์ละ 5 เท่า

- **Salesforce pay-per-resolution $2 = pricing model จุดใหม่ของทั้งอุตสาหกรรม agent** — CFO อ่านเลข TCO ง่ายขึ้น, procurement approve เร็วขึ้น, vendor รับ downside risk. คู่แข่งที่ยังคิด per-conversation/per-message จะโดน dismiss. **Builders**: pricing engine ต้องรองรับ outcome-based billing (attribution, resolution definition, dispute); framework token-based จะกลายเป็น commodity. **Business ในไทย** ที่ยังตัดสินใจไม่ได้กับ Agentforce เพราะกลัว TCO — pay-per-resolution ตัดข้ออ้างสุดท้ายทิ้งไป

- **SnapLogic MCP Builder GA = iPaaS reposition เป็น agent middleware** — MCP adoption จะไม่ pass ผ่าน developer สร้าง server มือทีละตัวอีก, แต่ผ่าน integration platform ที่มี install base พันตัวอยู่แล้ว. **Builders**: MCP server ที่เขียนมือจะเป็น minority, battle คือ "1,000 connector ของใครดีกว่า" + tool schema/description quality. **Business ในไทย** ที่ใช้ SnapLogic/MuleSoft/Boomi อยู่ — คุย vendor ตอนนี้เรื่อง MCP roadmap; pipeline ที่มีอยู่กำลังกลายเป็น agent-accessible ทันที ไม่ต้อง migration project

- **Straiker $64M = agent security กลายเป็น line item ที่ separate จาก endpoint/DLP/SIEM** — เพราะ agent สร้าง blast radius ที่เครื่องมือเดิมมองไม่เห็น (chain tool call 50+, cross-system identity, prompt injection surface). Pattern: founder จาก security incumbents (Palo Alto, Zscaler) กระโดดออกมาสร้าง layer ใหม่ที่ incumbents ทำเองไม่ทัน. **Builders**: framework agentic ต้องเปิด hook ให้ platform อย่าง Straiker inspect tool-call — vendor neutrality = table stakes. **Business ในไทย** (ธนาคาร ประกัน ที่มี SOC 2/PDPA obligation) — ก่อน deploy Claude Code / Copilot / Agentforce ในระดับ production, ให้ CISO เริ่ม "agent inventory" ทันที (audit shadow agent) แล้วค่อยเลือก monitoring vendor

- **Pattern หลักของ 4 ข่าวรวมกัน — ทุก layer ของ stack ถูกบีบให้ prove unit economics** — model layer (Cisco: routing > frontier), pricing layer (Salesforce: outcome > activity), tooling layer (SnapLogic: velocity > custom build), security layer (Straiker: agent-native > endpoint reuse). ครึ่งแรก 2026 พิสูจน์ "agent ทำงานได้"; ครึ่งหลังจะพิสูจน์ "agent คุ้มเงินได้อย่าง scale". **Signal ต่อ platform players**: ใครที่ยังเน้น demo capability / benchmark ranking — จะแพ้ให้ vendor ที่พูดภาษา CFO ได้ตรง

- Action ตรงสำหรับ **AI Agent Platform builders**:
  - (1) **Model routing เป็น first-class** — โมเดล default ของ framework ต้องเป็น mid-tier (Sonnet 5, Haiku 4.5, GPT-5.6 Terra/Luna); auto-escalate ไป frontier เฉพาะ intent ที่ตรวจ complexity แล้ว
  - (2) **Outcome-based pricing option** — เตรียม billing engine ให้ลูกค้าเลือก per-token / per-call / per-resolution; per-token-only จะโดน dismiss ใน RFP
  - (3) **MCP publish จาก asset ที่มีอยู่** — ถ้าเป็น iPaaS/API management, ปล่อย "one-click MCP" ภายในไตรมาสนี้ ไม่งั้นเสียตำแหน่งให้ SnapLogic
  - (4) **Agent runtime observability hook** — ต้องมี API ให้ Straiker/Prompt Security/Lakera inspect tool-call ก่อนถูก enterprise ใน short-list

- Action ตรงสำหรับ **Business / users** ที่ deploy AI Agent:
  - (1) **Model gateway ก่อน frontier subscription** — Cisco แสดงให้เห็น scale ระดับหมื่นคนต้องมี routing layer; ประเมิน Portkey / LiteLLM / Databricks AI Gateway ก่อน commit vendor sub
  - (2) **Re-run agent business case ทุกไตรมาส** — outcome-based pricing (Salesforce), cheaper mid-tier model (Sonnet 5, Terra/Luna), free tooling (SnapLogic MCP Builder) เปลี่ยน TCO ทุก 90 วัน
  - (3) **Insist self-hosted + audit hook** — vendor ที่ให้แค่ SaaS API endpoint แต่ไม่เปิด hook สำหรับ agent security = red flag governance ในปี 2027
  - (4) **CISO เริ่ม "agent inventory" ทันที** — ก่อน 1B agents deploy ปี 2029, เริ่มมี line item งบ "AI/agent security" ในปี 2027 = พอทัน; ปีถัดไปจะช้าเกิน
