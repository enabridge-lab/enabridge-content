# Daily Brief — 26-06-30 รอบ 17:14 (Bangkok)

## Theme ของรอบ

**สัปดาห์ 24-30 มิ.ย. 2026 = "วงการ AI agent กำลัง re-cast ทุก layer พร้อมกัน — model + silicon + distribution + protocol"**. 4 ข่าวรอบนี้ไม่ใช่ event แยกกัน แต่ประกอบเป็น mental model ของ enterprise AI stack ปลายปี 2026 ที่กำลังหา equilibrium ใหม่ — frontier model split เป็น 3 tier + government gate, silicon vendor เริ่มตี compiler/runtime moat ของ NVIDIA, hyperscaler distribution กำลัง commoditize, และ protocol layer ของ agent ปลด constraint horizontal scaling

**OpenAI GPT-5.6 Sol/Terra/Luna** (26 มิ.ย.) เป็น signal ใหญ่ที่สุดของรอบ — frontier model release ครั้งแรกที่ split เป็น 3 tier (Sol $5/$30, Terra $2.50/$15, Luna $1/$6) + gated ด้วย US frontier AI review process ของรัฐ (~20 partner approved). Sol อ้าง agentic improvements ใน coding/biology/cybersecurity + safety stack แข็งสุด. เป็นครั้งแรกที่ Washington governance เข้ามาเป็น distribution gate ของ commercial AI model — และ Anthropic Mythos 5 ก็โดน gate เดียวกัน (เพิ่งปลดล็อก partial เมื่อ 27 มิ.ย.). โลก frontier AI กลายเป็น dual-track: commercial GA + government-vetted preview

**Qualcomm × Modular** (24 มิ.ย.) คือ first attack บน CUDA moat ใน 18 ปี. Qualcomm จ่าย **$3.92B all-stock** ซื้อบริษัทของ Chris Lattner ผู้สร้าง LLVM/Swift — Modular เป็นเจ้าของ Mojo language + MAX inference engine ที่ port AI model ระหว่าง NVIDIA/AMD/Intel/Apple Silicon/Qualcomm ได้ด้วย API เดียว. Bloomberg + Reuters รั่วก่อนหน้าว่า Qualcomm กำลังเจรจาซื้อ Tenstorrent (Jim Keller) เพิ่มอีก **$8-10B** — ถ้าปิดจริง Qualcomm ประกอบ full-stack ~**$14B alternative** ตรงเข้าตลาด inference $255B. เป็น software acquisition ที่ใหญ่ที่สุดในประวัติ Qualcomm และเป็น signal ว่า moat ของ NVIDIA เริ่มมีรอย

**Claude on Microsoft Foundry GA** (ปลาย มิ.ย.) คือ Anthropic ยอม pivot จาก vendor-tight strategy เป็น distribution maximalism. Opus 4.8 + Haiku 4.5 เข้า Foundry แบบ GA — รันบน **NVIDIA GB300 Blackwell Ultra** ที่ Azure จัดให้ พร้อม Entra ID + Azure RBAC + governance. ครั้งแรกที่ Anthropic ยอมให้ inference workload วิ่งนอก in-house เต็มตัว. Microsoft ได้ Claude เป็น OpenAI alternative ใน Foundry ตรง — ลูกค้า Azure route call ระหว่าง GPT-5.6, Claude Opus 4.8 ผ่าน API เดียวได้, switching cost ใกล้ศูนย์. Anthropic ตัดสินใจว่า scale > control

**MCP 2026-07-28 stateless RC** (21 พ.ค. lock, 28 ก.ค. finalize) เป็น protocol-level maturity ที่สำคัญที่สุดในรอบ 2 ปี. Mcp-Session-Id header + protocol session ถูกตัดออก — request ทุกประเภท land ที่ server instance ไหนก็ได้, autoscaling เป็น vanilla. เพิ่ม **Tasks extension** สำหรับ long-running work, **MCP Apps** สำหรับ server-rendered UI ที่ client render ได้ทุก vendor, **OAuth-aligned authorization**. มาในจังหวะที่ MCP ครองแล้ว: ~**97M SDK downloads/เดือน**, **9,400+ public servers**, native support ทุก major lab — และ Anthropic เพิ่งเปิด MCP Tunnels พ.ค. ทำให้ MCP เข้า enterprise infra เต็มตัว

Pattern หลักของ 4 ข่าวรวมกัน — **ทุก layer ของ AI Agent stack กำลัง re-architect พร้อมกัน**. Model layer เปลี่ยนจาก "ตัวเดียวใช้ทุกอย่าง" เป็น "3 tier + gate". Silicon layer เปลี่ยนจาก "NVIDIA monopoly via CUDA" เป็น "vendor-portable inference via Modular MAX". Distribution layer เปลี่ยนจาก "lab control" เป็น "hyperscaler gateway". Protocol layer เปลี่ยนจาก "stateful session" เป็น "stateless HTTP-native". ผู้ชนะของไตรมาสนี้ — บริษัทที่ position ตัวเองข้าม layer (Anthropic ใน model + distribution + protocol; Microsoft ใน distribution + Foundry; Qualcomm ใน silicon + compiler). **Agent platform builders** ที่อยู่ในชั้น integration backbone (LangChain, LlamaIndex, Vercel AI SDK, Dust, n8n, OpenBridge, ทุกคน) ต้องตัดสินใจไม่ยอม commoditize — pivot เป็น multi-cloud orchestrator ที่ route across Foundry/Bedrock/Vertex/on-prem + MCP-native + tier-aware + vendor-portable inference

## TL;DR

- **OpenAI Sol/Terra/Luna คือ first explicit acknowledgment ว่า frontier release จากนี้ไม่ใช่ "ของใหม่ใหญ่หนึ่งตัว" แต่เป็น product line** — โลกที่เคยมี GPT-4 แล้วทุกคนใช้เหมือนกันหมดจบแล้ว. ทุก agent platform procurement ต้องตอบ "ใช้ tier ไหนสำหรับ workload ไหน" ก่อน vendor. Government-gate เข้ามาเป็น product feature — แบ่ง customer tier ได้ก่อนคู่แข่ง + เปิด channel ใหม่กับรัฐ + ลด liability. **Agent platform builders** (LangChain, Vercel AI SDK, Dust, n8n, Cloudflare AI Gateway, ทุกคนที่ทำ orchestration) ต้องออก "tier-aware routing" ภายในไตรมาสนี้. **Business ที่ใช้ agent** ได้ pricing leverage ใหม่ — workload เดียวรัน Luna ราคา 1/30 ของ Sol ได้ ถ้าออกแบบ prompt + retrieval ดี

- **Qualcomm-Modular $3.9B คือ first attack บน CUDA moat ใน 18 ปี** — ดีลนี้ confirm ว่ามันเป็น software acquisition ไม่ใช่ hardware deal. Modular MAX ที่ Lattner สร้างคือ runtime ตัวเดียวที่ produce-grade เทียบเคียง CUDA ได้ — และตอนนี้ Qualcomm เป็นเจ้าของ. รวมกับ Tenstorrent ที่กำลังจะปิด = $14B full-stack alternative ใน inference market $255B. NVIDIA จะต้องตอบ — และไม่ว่าตัดสินใจอะไร moat ของ CUDA จะรั่วเป็นครั้งแรกตั้งแต่ปี 2008. **Agent platform builders ต้อง assume vendor-portable inference เป็น default** ภายใน 12 เดือน — connector "Modular MAX backend" + "Qualcomm Cloud AI 100" ออกก่อนได้ moat. **Business ที่เลือก agent platform** ต่อจากนี้ต้องถาม "platform นี้รัน inference ผ่าน vendor ไหนได้บ้าง"

- **Claude on Foundry GA คือ Anthropic ยอมเลิก control เพื่อ distribution** — เดิม Anthropic positioning เป็น Apple ของ AI: premium, narrow, vertically integrated. ตอนนี้ pivot — Opus 4.8 + Haiku 4.5 รันบน Azure GB300 เต็มตัว, Entra ID + Azure RBAC + governance. Microsoft ได้ Claude เป็น OpenAI alternative ใน Foundry. Business ที่ใช้ Azure route call ระหว่าง GPT-5.6, Claude ผ่าน API เดียว — switching cost ใกล้ศูนย์. Early warning ของ enterprise AI commoditization. **Agent platform builders** ต้อง integrate Foundry channel เป็น first-class (default endpoint pattern สำหรับลูกค้า Azure stack) + pivot positioning เป็น "multi-cloud orchestrator + governance layer" ก่อน hyperscaler gateway กิน horizontal

- **MCP 2026-07-28 stateless RC คือ protocol-level maturity ครั้งใหญ่สุดของ MCP** — Mcp-Session-Id หาย, autoscaling เป็น vanilla, ต้นทุน operate MCP server fleet ลด 60-80%. Tasks extension แก้ long-running work pattern, MCP Apps เปิด UI sandbox cross-vendor, OAuth-aligned auth ปลด enterprise procurement. ตลาดอยู่ที่ ~97M SDK downloads/เดือน + 9,400+ public servers — MCP เป็น de facto standard แล้ว. **Agent platform builders มี window สั้น**: upgrade SDK ทั้งหมดเป็น 2026-07-28-compatible ภายในสัปดาห์ spec finalize, implement Tasks extension เป็น first-class, build MCP Apps preview ก่อน Cloudflare/Vercel ยึดตลาด. **Business** ที่กำลังจะ wrap internal API เป็น MCP server ได้ deployment ง่ายขึ้น 5-10 เท่า

- **Pattern หลักของ 4 ข่าวรวมกัน — ทุก layer ของ AI Agent stack กำลัง re-architect พร้อมกัน**. Model split. Silicon vendor-portable. Distribution commoditize. Protocol stateless. ผู้ชนะของไตรมาสนี้ = บริษัทที่ position ตัวเองข้าม layer. Agent platform ที่อยู่ในชั้น integration backbone (LangChain, LlamaIndex, Vercel AI SDK, Dust, n8n, OpenBridge, ทุกคน) ต้องตัดสินใจชัด — ไม่ยอม commoditize, ขยับขึ้นเป็น **multi-cloud + tier-aware + MCP-native + vendor-portable** gateway ที่ Foundry/Bedrock/Vertex/on-prem คนเดียวให้ไม่ได้

- Action ตรงสำหรับ **AI Agent Platform builders**:
  - (1) **"Tier-aware routing"** — feature route ระหว่าง Sol-class, Terra-class, Luna-class workload (และ Opus/Sonnet/Haiku ของ Anthropic) ตาม cost/latency/data residency rule — ออกภายในเดือน
  - (2) **"Modular MAX backend"** connector — รัน inference engine แทน NVIDIA-direct call ภายใน Q3 — เป็น differentiation ก่อน Qualcomm-Modular ปิดดีลครึ่งหลังปี
  - (3) **"Foundry-aware default"** + **"Bedrock-aware default"** + **"Vertex-aware default"** — preset endpoint สำหรับลูกค้าแต่ละ cloud stack — checkbox สำหรับ procurement
  - (4) **MCP 2026-07-28 compliance** — upgrade SDK เป็น stateless-compatible ภายในสัปดาห์ spec finalize (28 ก.ค.) + implement Tasks extension เป็น first-class
  - (5) **"Multi-cloud orchestrator + governance"** positioning — vendor-neutral router ที่ route ระหว่าง Foundry/Bedrock/Vertex/on-prem + policy enforcement + PII redaction + audit ที่ครอบทุก cloud
  - (6) **Regional sovereign inference partnership** — Qualcomm SEA, NECTEC, Singapore IMDA, อินโด, เวียดนาม — สร้าง local integration layer ระหว่าง Modular MAX + local datacenter + local API (bank, government, hospital) ก่อน global vendor ลงมาแข่ง

- Action ตรงสำหรับ **Business / users** ที่ deploy AI Agent:
  - (1) **Audit ค่า inference**: ดู workload ที่ใช้ flagship model — ส่วนใหญ่ route ไป mid/low tier ได้โดยไม่เสีย quality. Sol→Luna อาจประหยัด 30x
  - (2) **ก่อน sign contract กับ agent platform** ต่อจากนี้ ต้องถาม 4 คำถาม: support tier-aware routing ไหม? support multi-cloud (Foundry+Bedrock+Vertex) ไหม? support MCP 2026-07-28 ไหม? Modular MAX backend ทำได้ไหม?
  - (3) **Wrap internal API เป็น MCP server** — stateless transition ทำให้ง่ายขึ้น 5-10 เท่า. เริ่ม pilot กับ core system 1 ตัว (CRM, ticketing, internal data) ภายใน Q3
