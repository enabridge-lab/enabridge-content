# OpenAI x PwC สร้าง "AI-native finance function" — dogfood ใน OpenAI ก่อน scale ขายลูกค้า

## TL;DR
- PwC + OpenAI ประกาศ **first-of-its-kind AI-native finance function** — vertical agent ครอบ planning, forecasting, reporting, procurement, payments, treasury, tax, close
- OpenAI **ใช้ procurement agent ใน finance org ของตัวเองก่อน** — เก็บ learning จาก dogfood แล้วค่อยขายผ่าน PwC delivery network
- Counter-strategy ต่อ Anthropic — Anthropic ขายผ่าน vertical software stack (Moody's, FIS), OpenAI ขายผ่าน **Big-4 services partner** ที่คุม CFO relationship

## เกิดอะไรขึ้น

วันที่ 5 พฤษภาคม วันเดียวกับที่ Anthropic ประกาศ Wall Street push, **OpenAI เปิดความร่วมมือกับ PwC** เพื่อสร้างสิ่งที่สองค่ายเรียก "first-of-its-kind OpenAI Native Finance Function" — vertical agent stack ที่ครอบ workflow ของ CFO office จริง: financial planning, forecasting, reporting, procurement, payments, treasury, tax, accounting close — ครบ 8 surface ของ finance org ใน enterprise

โครงสร้างของดีลที่แตกต่างจากปกติ: **OpenAI สร้าง procurement agent ตัวแรกใน OpenAI's own finance organization** — ใช้ engineer ของตัวเองและ PwC ทีม forward-deployed ฝังที่ออฟฟิศ OpenAI เพื่อ build agent ที่จัดการ vendor onboarding, contract review, PO routing, invoice matching, spend analysis ของ OpenAI เอง — แล้วเอา learning จาก dogfood ไป generalize เป็น product ที่ PwC ขายต่อให้ลูกค้า Fortune 1000

PwC เคลม model นี้ "moves finance beyond task automation" — หมายถึงไม่ใช่ RPA ที่ replace ปุ่มกด แต่ agent ที่ "execute complex multi-step finance work" และ "collaborate across workflows" ในวง finance language ของ Big 4: นี่คือ **transformation engagement ที่ OpenAI ใส่ technology + PwC ใส่ change management** — typical PwC fee scale $5–50M ต่อ engagement, contract length 18–36 เดือน

OpenAI เคลมว่าใน new operating model "**finance professional ย้ายจากการ execute → supervise/govern/improve agent over time**" — ซึ่งเป็น repositioning หลังจากที่ AI ถูกกล่าวหามายาวว่าจะแทน finance headcount ในวง CFO ของ Fortune 500 messaging นี้สำคัญเพราะ procurement decision อยู่กับ CFO เอง ที่ต้องเซ็นเขียน budget ของตัวเองให้ลด

## ทำไมสำคัญ

นี่คือ **คู่ตรงข้ามทางยุทธศาสตร์ของ Anthropic Wall Street day** — ทั้งสองค่ายเลือก vertical "financial services" เหมือนกัน แต่ approach ต่างกันสุดขั้ว Anthropic เลือก **horizontal play across financial services** — banking + asset management + insurance — ผ่าน vertical software stack (Moody's, FIS, Microsoft 365); OpenAI เลือก **vertical play within enterprise finance function** — CFO office โดยเฉพาะ — ผ่าน Big 4 services delivery (PwC) ที่มี CFO relationship อยู่แล้ว

เปรียบ unit economics: **Anthropic margin ดีกว่า** เพราะไม่ต้องแบ่ง fee กับ services firm ใหญ่ — Moody's และ FIS เป็น software vendor partner ที่ revenue share น้อยกว่า; **OpenAI scale เร็วกว่า** เพราะ PwC มี salesforce 364,000 คนทั่วโลก, มี CFO relationship ใน Fortune 500 ระดับ 90%+, มี delivery capacity 2–3 ลำดับขนาดเหนือ Anthropic Applied AI team — ในระยะ 12–18 เดือน OpenAI จะมี logo ใน CFO office ของ Fortune 500 เร็วกว่า Anthropic

จุดที่เกมจริงคือ **Big 4 ทุกรายกำลังจะถูกบีบให้เลือกข้าง** — Deloitte ผูกกับ NVIDIA + Google Cloud อยู่แล้ว; KPMG ใช้ Microsoft 365 Copilot เป็นหลัก; EY signal Anthropic; **PwC + OpenAI = Wall Street alignment ระดับ services tier** — ลูกค้า Fortune 500 ที่ใช้ PwC เป็น advisor หลักจะถูก default ไปทาง OpenAI ในรอบ procurement หน้า ตรงข้ามกับลูกค้าที่ใช้ EY ที่ default ไป Anthropic — pattern เก่าของ "Oracle shop vs SAP shop" กำลังเกิดใหม่ที่ frontier model layer

Signal ระยะกลาง: **dogfood-first product development** กลายเป็นนาฬิกาวัด credibility ของ vertical agent — ถ้า OpenAI build procurement agent ใน OpenAI's own finance org แล้ว ship ไม่ทันลูกค้า — credibility ของทั้ง dealจะหาย; ถ้า ship ทันและ generalizable — Anthropic จะถูกกดดันให้ release case study "Claude inside Anthropic's own X function" ตามใน 2 ไตรมาสถัดไป

## มุม OpenBridge

**สำหรับ OpenBridge ดีลนี้บอกว่า "CFO office" คือ vertical แรกที่ AI agent เริ่ม move จาก experimental → production budgeted line item** — procurement, AP/AR, expense management ทั้งหมดเป็นพื้นที่ที่ workflow มีโครงสร้างชัด, integration หลายระบบ (ERP + procurement + bank + tax filing + audit), และ ROI วัดได้เป็น cost-per-invoice หรือ days-payable-outstanding — โครงสร้างที่ OpenBridge ออกแบบมาจัดการอยู่แล้ว

Action ตรง: (1) **map OpenBridge flow library กับ 8 surface ของ CFO** — ระบุว่าโครงสร้าง integration pattern ของ OpenBridge support workflow ไหนของ procurement-to-pay, order-to-cash, record-to-report ได้ทันที; ทำ public catalog "AI-ready finance flows on OpenBridge" (2) **dogfood case study OpenBridge เอง** — copy pattern OpenAI/PwC: build agent ที่ automate workflow ภายในของ OpenBridge เอง (เช่น customer billing reconciliation, partner commission calc) แล้ว publish learnings เป็น marketing — credibility สูงกว่าทุก case study ของลูกค้า (3) **ระวัง positioning** — ถ้า OpenBridge อยู่ระหว่าง CFO กับ ERP, อย่า pitch เป็น "Workato/Zapier-but-AI" — pitch เป็น "agent execution layer for finance ops" ที่ governance + observability เป็นชั้นแรก — ราคาขึ้นได้อีก tier
