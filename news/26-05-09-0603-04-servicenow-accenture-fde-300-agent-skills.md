# ServiceNow + Accenture เปิด FDE สำหรับ agentic AI — 300+ pre-built skills + AI Control Tower เป็น "command center" ของ agent

## TL;DR
- 6 พ.ค. 2026 ที่งาน **Knowledge 2026** ServiceNow + Accenture เปิด **Forward Deployed Engineering (FDE) program** ร่วม — engineer ของทั้งสองค่าย "embed" ลงใน customer environment พร้อมกัน, build agentic workflow native บน ServiceNow AI Platform, deliver value ใน production **ก่อน** rollout เต็มบริษัท
- ลูกค้าเข้าถึง **300+ pre-built AI agent skills** + agentic workflow บน ServiceNow AI Platform — Accenture overlay industry depth (financial services, telecom, healthcare); ServiceNow ขาย platform reach (8,100+ ลูกค้า enterprise ที่มี ServiceNow ใบ)
- **AI Control Tower** = unified command center ที่ govern, secure, manage agent at scale — เห็น performance + outcome ทุก agent โดยไม่ trade speed for control; ServiceNow position ตัวเองชัดเจนว่าเป็น "control plane" เหนือ agent stack

## เกิดอะไรขึ้น

วันที่ 6 พ.ค. 2026 ที่ Knowledge 2026 (annual conference ของ ServiceNow ที่ Las Vegas) ServiceNow และ Accenture ประกาศ **Forward Deployed Engineering program** ร่วม — model ที่ลอกมาจาก Palantir + OpenAI/Anthropic services arm ที่เพิ่ง launch สัปดาห์เดียวกัน วิธีทำงาน: ทีม FDE ของ ServiceNow (ที่เป็น AI-native) + ทีม FDE ของ Accenture (ที่เป็น industry expert) เข้าไป **embed ใน customer environment พร้อมกัน** — ไม่ใช่ remote consulting แต่นั่งทำงาน inside ระบบลูกค้า build agentic workflow native บน ServiceNow AI Platform deliver outcome ใน production ก่อน rollout เต็ม pattern คือ "show value first, scale second"

ลูกค้าได้สอง asset หลัก หนึ่งคือ **300+ pre-built AI agent skills + agentic workflow** ที่ ServiceNow ship เป็น marketplace — แต่ละ skill แก้ขั้นตอน specific เช่น "auto-resolve ticket level 1", "reconcile invoice mismatch", "onboard new vendor", "rotate access credential" — Accenture เพิ่ม **industry vertical** อีกชั้น (banking AML, telecom outage triage, hospital bed assignment) ทำให้ enterprise ที่เข้า program ไม่ต้อง start from zero สองคือ access สู่ **AI Control Tower** ที่ ServiceNow GA ไปไตรมาสก่อน — unified pane ที่ govern agent identity, permission, observability, kill-switch ทุกตัวในองค์กร โดย vendor-agnostic (claim ว่ารองรับ agent จาก Salesforce, Microsoft, OpenAI, Anthropic, custom)

ที่น่าจับตาในการ frame ของ ServiceNow คือคำว่า "**from pilot to production at scale**" — ทุก quarterly survey ที่ผ่านมา (Gartner, IDC, Salesforce State of AI) บอกตรงกันว่า ~70-80% ของ agent project ค้างที่ POC stage ไม่ขึ้น production ServiceNow + Accenture pitch FDE ว่าเป็น "antidote to POC purgatory" — Accenture ขยับจาก consultant ที่เขียน slide เป็น engineer ที่ deliver workflow รัน production จริง CEO ของ ServiceNow Bill McDermott bo ใน keynote ว่า program ตั้งเป้า "**measurable business outcomes: faster operations, lower costs, improved customer experiences**" — ไม่มีตัวเลข deal size, ไม่มี named pilot customer ที่ public แต่ Accenture pre-announce ว่ามี "leading global financial services" และ "Fortune 100 telecom" อยู่ใน initial cohort

## ทำไมสำคัญ

ข่าวนี้คือ **third example** ในรอบสองสัปดาห์ของ pattern เดียวกัน — OpenAI ตั้ง enterprise services arm $4B (4 พ.ค.), Anthropic ตั้ง Blackstone-backed services co (5 พ.ค.), ตอนนี้ ServiceNow + Accenture (6 พ.ค.) ทุกคนยอมรับสิ่งเดียวกัน: **product-led growth ไม่ scale ในตลาด agentic enterprise** เพราะ workflow ขององค์กรซับซ้อน, integration เยอะ, change management ใหญ่ ลูกค้าซื้อ agent platform แล้วใช้ไม่เป็น — ต้องมี "human + AI hybrid team" เข้าไปทำให้รัน Palantir รู้เรื่องนี้ตั้งแต่ 2010 และ 16 ปีต่อมา playbook ของพวกเขาเพิ่งกลายเป็น mainstream

จุดที่ ServiceNow ได้เปรียบกว่า OpenAI/Anthropic services arm คือ **distribution** — ลูกค้า 8,100 รายมีใบ ServiceNow อยู่แล้ว (ITSM, HRSD, CSM) มี data ที่ปลด-friction การ build agent workflow ทันที agent ใหม่ที่ฝัง ใน ServiceNow มี context ของ ticket history, change request, vendor contract, employee record อยู่แล้ว — ไม่ต้อง bootstrap data layer ส่วน OpenAI/Anthropic services arm ต้องเริ่มจาก scratch เป็นเดือนทุก deal Accenture เป็น GTM partner ที่ยอมเอา talent มา embed (ไม่ใช่ flip ลูกค้าไป Microsoft Azure/Google Cloud) เป็นการ commit ที่หนักมากของ Accenture ซึ่ง pivot ตัวเองจาก strategy → managed services → "agentic engineering firm" ในรอบ 4 ปี

ที่ตลาดควรจับตา: **AI Control Tower** จะกลายเป็น flashpoint — ServiceNow claim ว่าเป็น control plane เหนือ agent stack แต่ Microsoft Agent 365 ก็ claim, Salesforce Agentforce 360 ก็ claim, ChatGPT Frontier ก็ claim, IBM watsonx Concert ก็ claim ลูกค้า Fortune 500 จะไม่อยากใช้ control tower 5 ตัว — การ choose จะเกิดในปลายปี 2026 อย่างช้า ServiceNow มีข้อได้เปรียบที่อยู่ใน workflow ของ IT/HR ลึกที่สุด แต่ Microsoft อยู่ใน productivity layer ใหญ่กว่า — เกมยังเปิดอยู่

## มุม OpenBridge

OpenBridge ต้องอ่านสามอย่างจากข่าวนี้ **(1) ServiceNow AI Platform คือ surface สำคัญที่ MCP connector ต้องไปอยู่** — 300+ agent skills ของ ServiceNow ส่วนใหญ่เรียก system ภายนอก (SAP, Workday, Salesforce, Stripe) ผ่าน connector layer; OpenBridge ที่ ship MCP server ที่ register ใน ServiceNow Skill marketplace จะ inherit traffic ของ Fortune 500 cohort ทันที — pattern คล้าย ที่ Stripe/Twilio plugin Salesforce AppExchange ในยุค 2015 — distribution channel ที่ต่อยากแต่ revenue compound เร็ว

**(2) AI Control Tower telemetry = ตลาด commodity ที่ OpenBridge เข้าได้** — ทุก connector call ของ OpenBridge ต้อง emit event schema ที่ Control Tower ingest ได้: action ID, agent identity, target system, latency, cost, outcome (success/fail/escalate); ถ้า OpenBridge build SDK ที่ wrap MCP call ให้ emit telemetry ไป ServiceNow + Microsoft Purview + Splunk + Datadog พร้อมกัน → enterprise admin set policy ได้กลาง ๆ — กลายเป็น default integration layer

**(3) FDE model = playbook ที่ OpenBridge ควรเลียนสำหรับ Thai/SEA enterprise** — ลูกค้า enterprise ในไทยที่อยาก deploy agent มักจ้าง consulting (Accenture, Deloitte, KPMG) แต่ consultant ไม่มี deep MCP/connector expertise OpenBridge สามารถ position "embedded engineer" ที่นั่งกับ consultant 2-4 สัปดาห์ต่อ deal, ปลด integration friction, deliver workflow ใน sprint แรก — ราคา premium แต่ unlock deal size 10x compared to self-serve subscription; เป็น playbook ที่ scale จาก 50 → 500 ลูกค้าใน 18 เดือน
