# Daily Brief — 26-05-15 รอบ 06:05 (Bangkok)

## Theme ของรอบ

**สัปดาห์ที่ 13–14 พ.ค. 2026 คือ "Distribution-channel war" ของ frontier lab** — ข่าวใหญ่ 4 ข่าวบอกตรงกันว่า value pool ของ AI ไม่ได้อยู่ที่ "ใครมีโมเดลฉลาดที่สุด" แต่อยู่ที่ **"ใครยึด distribution channel ของ enterprise stack ได้ก่อน"** — ERP, voice infrastructure, workspace, SMB tooling — ทุกชั้นถูก lock-in ไปคนละทิศ ที่กรุงเทพเช้านี้ **SAP Sapphire 2026** เป็น centerpiece — Christian Klein เปิดวิสัยทัศน์ **Autonomous Enterprise** ที่ฝัง agent ลง core ERP: **50+ Joule Assistants** ใน finance/supply chain/HR/procurement สั่งงาน **200+ specialized agents** เข้าถึง **7 ล้าน data field** ผ่าน SAP Business AI Platform; **Anthropic** ขยาย partnership ทำให้ **Claude เป็น primary reasoning engine** ทั่ว portfolio — เปิดทาง Claude เข้าถึงลูกค้า SAP "หลักแสนราย" ทันที; **Palantir** เป็นขาที่สาม ทำ agentic migration toolkit ลด effort 35% (GA Q3); proof point: **Autonomous Close Assistant** บีบ financial close จากหลายสัปดาห์เหลือหลายวัน — Forrester ออก review ว่า "credible แต่ concentration risk สูง" (เดิมพันกับ Anthropic ทั้งหมด)

วันที่ 12 พ.ค. **Vapi** (voice AI platform จาก SF) ปิด **Series B $50M** valuation **~$500M** lead by Peak XV + Microsoft M12, Kleiner Perkins, Bessemer; headline ที่ทำให้วงการอึ้ง: **Amazon Ring โอน 100% ของ inbound customer call** มาวิ่งบน Vapi หลังทดสอบ vendor 40+ ราย — deploy zero-to-production ใน **2 สัปดาห์**; Vapi แตะ **1B calls สะสม**, รัน **1–5M calls/วัน**, ARR "healthy 8 figures"; เลือก build vertical stack เอง (telephony + STT + LLM routing + TTS, latency < 500ms) ไม่ใช่ wrapper OpenAI Realtime — เหตุผลที่ Ring เลือกเหนือ Twilio/ElevenLabs/Bland/Retell/Synthflow; signal ตลาด voice AI ผ่าน threshold production แล้ว enterprise vertical อื่นจะตามทันที 2 ไตรมาสนี้

วันที่ 13 พ.ค. **Notion** เปิด **Developer Platform** ใหม่ที่กระโดดจาก note-taking tool ขึ้นเป็น agentic workspace 3 ขา — **Workers** (hosted code runtime), **External Agent API** (เชื่อม agent ภายนอก partner วันแรก Claude Code/Codex/Cursor/Decagon), **Database Sync** (bi-directional กับ Salesforce/Zendesk/Postgres); positioning ใหม่ของ Notion = "control room ที่ agent ค่ายอื่นมา dock" ไม่ใช่ destination ที่ user มาพิมพ์ doc; ticket ใน support route ไป coding agent → propose fix → loop human approve ใน Notion page เดิม; pricing ฟรี beta แล้วเริ่มเก็บผ่าน Notion credits 11 ส.ค.; first-mover ของ workspace orchestration ก่อน Slack/Atlassian/Microsoft

ฝั่ง market structure — **Ramp AI Index** ของเดือน พ.ค. เผย: เม.ย. **Anthropic = 34.4%** ของธุรกิจ US ที่ใช้ AI vendor / **OpenAI = 32.3%** — Anthropic แซงเป็นครั้งแรกในประวัติศาสตร์; Anthropic โต 4 เท่าใน 12 เดือน (7.94% → 34.4%) ขณะ OpenAI โตแค่ 0.3pp; driver = **Claude Code** (product โตเร็วที่สุดในประวัติ Anthropic, ~70% ของ Fortune 500 engineering team ที่ใช้ coding agent ใช้ Claude Code); 14 พ.ค. เปิด **Claude for Small Business** + workshop tour ฟรี 10 เมือง US — ลงไปจับ SMB ที่ OpenAI ทิ้งเปล่า; Anthropic ใส่ credit meter ใหม่บน outside agent tool ปกป้อง margin OpenAI ตอบโต้ด้วยขยาย Codex + ลดราคา agent token — 2 ค่ายเลือก strategy ตรงข้ามใน inflection ลึก = ตลาดยังไม่ converge ลูกค้าจะมี leverage สูง 6–12 เดือนข้างหน้า

Pattern หลักประจำสัปดาห์: **frontier lab ไม่ขายของผู้ใช้ตรงแล้ว — ฝังตัวเข้า distribution channel ของ enterprise stack ที่มี user อยู่แล้ว** Anthropic เลือก SAP เป็นช่อง ERP, Notion เป็นช่อง workspace, QuickBooks/Shopify เป็นช่อง SMB; Claude กลายเป็น "primary brain" ใต้ทุกแอปที่ enterprise ใช้ ผู้ใช้ไม่ต้องรู้ด้วยซ้ำว่าใช้ Claude; pattern นี้คล้าย Intel Inside ปี 1990s + Stripe pattern ปี 2010s — value extract จาก ubiquity ไม่ใช่ brand awareness. OpenBridge ต้องเลือกข้าง: เป็น **"protocol layer ที่ agent ทุกค่ายมาเชื่อม"** (Notion-like, vendor-neutral) หรือ **"deep vertical integration กับ Claude ที่ Anthropic global ไม่ลงมา"** (SEA-localized, language-first) — ทั้ง 2 ทางมีอนาคต แต่ต้องเลือก เพราะ middle ground ระหว่าง "horizontal generic" กับ "vertical deep" จะถูกบีบหายไประหว่าง frontier lab + SI ใหญ่

## TL;DR สำหรับ Yoh

- **Frontier lab ฝังเข้า enterprise stack ผ่าน distribution channel — Anthropic แซง OpenAI ครั้งแรกในตลาด US enterprise** SAP Sapphire 2026 เปิด Autonomous Enterprise ฝัง Claude เป็น primary reasoning engine ทั่ว portfolio AI เข้าถึง customer SAP "หลักแสนราย" ทันที; Notion เปิด External Agent API ที่ Claude Code/Codex/Cursor/Decagon มา dock; Anthropic เปิด Claude for Small Business ฝัง QuickBooks/Shopify/HubSpot; pattern = Intel Inside ปี 1990 + Stripe ปี 2010 — value extract จาก ubiquity ไม่ใช่ brand awareness; Ramp AI Index ยืนยันชัด Anthropic ครอง 34.4% ของธุรกิจ US แซง OpenAI 32.3% เป็นครั้งแรกในประวัติ; Anthropic โต 4 เท่าใน 12 เดือน OpenAI โต 0.3pp เท่านั้น
- **Voice AI ผ่าน production threshold แล้ว — Vapi 1B calls + Amazon Ring 100% inbound** ก่อนหน้านี้ voice AI ติดที่ pilot 30% deflection การที่ Ring (Amazon-owned, regulatory pressure) ยอมเอา 100% inbound มาให้ AI ดูคือ signal ที่ enterprise vertical อื่นจะตามทันที 2Q นี้; Vapi ชนะ vendor 40+ รายเพราะ vertical stack build เอง (telephony + STT + LLM routing + TTS latency < 500ms) ไม่ใช่ wrapper; deploy ใน 2 สัปดาห์ = speed-to-production เป็น moat ใหม่ — ลูกค้า enterprise ไม่ตัดสินใจที่ feature list แล้ว
- **Workspace กลายเป็นสมรภูมิ orchestration ใหม่ — Notion เปิดทาง agent ของทุกค่ายมา dock ก่อน Slack/Atlassian/Microsoft** ticket → route ไป coding agent → propose fix → loop human ใน Notion page; positioning = control room ไม่ใช่ destination; แต่ credit-based pricing อาจ disintermediate ตัวเองถ้า heavy user bypass ไปใช้ Claude Code ตรง — pattern เดียวกับที่ Zapier ติด; signal ตลาด: **bi-directional sync** กลายเป็น standard, buyer mid-market ถามว่า "sync 2-way ได้ไหม conflict resolution ยังไง" ไม่ใช่ "เชื่อมกี่ระบบ"
- **ตลาด LLM enterprise ยังไม่ converge — Anthropic vs OpenAI เลือก strategy ตรงข้ามใน inflection ลึก** Anthropic ใส่ credit meter ปกป้อง margin / OpenAI ลดราคา agent token เพื่อดูดผู้ใช้กลับ; ลูกค้า enterprise/SMB จะมี leverage สูง 6–12 เดือนข้างหน้า; **single primary model** กลายเป็น default เลือกของ CIO ไม่ใช่ multi-model routing — pattern นี้ทำให้ integration platform ที่ขาย "เลือก model ได้" ต้อง reframe เป็น "model-aware workflow" ไม่ใช่ "model agnostic"
- Action ตรงสำหรับ OpenBridge: (1) **เลือกข้าง positioning ชัด** — เป็น "protocol layer vendor-neutral" (Notion-like) หรือ "deep vertical Claude-native สำหรับ SEA" — middle ground จะถูกบีบ ในอีก 12 เดือน (2) **ชู "Claude-native + MCP-first"** เป็น default narrative — Anthropic donate MCP ให้ Linux Foundation ธ.ค. 25 + AWS MCP Server GA 6 พ.ค. = MCP คือ de facto standard ของ agent-to-tool integration แล้ว (3) **build "vertical workflow ที่ Anthropic global ไม่ลงมา"** — ภาษาไทย, ระบบบัญชี Express/AccountStar, integration กับ LINE/Lazada/Shopee, ระบบ ERP ท้องถิ่น (Sapphire 2026 + Claude for Small Business จะ commoditize integration ใน 12 เดือน) (4) **Voice AI vertical SEA** — Vapi global ไม่ optimize ภาษาไทย/อินโด/เวียดนาม TTS quality ยังเป็น gap; partner กับ telco/contact-center SEA build localized voice agent platform ที่ accuracy > 95% (5) **Metric ที่ขาย enterprise buyer ต้องเปลี่ยน** — จาก "รองรับ X connector" เป็น **"go-live ภายในกี่วัน + zero-config use case กี่ตัว + bi-directional sync + conflict resolution"** — Ring เลือก Vapi เพราะ 2 สัปดาห์ ไม่ใช่ราคา/feature
