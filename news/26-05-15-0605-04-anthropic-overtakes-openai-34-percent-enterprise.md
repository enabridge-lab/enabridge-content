# Anthropic แซง OpenAI ครั้งแรกในตลาด enterprise — 34.4% vs 32.3% และเปิด Claude for Small Business ตามทันที

## TL;DR
- 13 พ.ค. — **Ramp AI Index** (วัดจาก spending pattern ของ 50,000+ ธุรกิจ US) เผย: เดือน เม.ย. **Anthropic = 34.4%** ของธุรกิจที่ใช้ AI vendor / **OpenAI = 32.3%** — Anthropic แซงเป็นครั้งแรก
- Anthropic โต 4 เท่าใน 12 เดือน (เม.ย. 25 = 7.94% → เม.ย. 26 = 34.4%) ขณะ OpenAI โตแค่ 0.3pp; driver หลักคือ **Claude Code** ซึ่งเป็น product โตเร็วที่สุดในประวัติ Anthropic
- 14 พ.ค. — Anthropic ประกาศ **Claude for Small Business** พร้อม workshop tour ฟรี 10 เมืองทั่ว US (100 entrepreneur/เมือง); จังหวะเดียวกัน OpenAI ตอบโต้ด้วยการขยาย Codex tooling + เปิด credit meter ใหม่บน Claude Code ที่ทำให้ราคา agent tool ปรับขึ้น

## เกิดอะไรขึ้น

วันที่ 13 พ.ค. 2026 Ramp (corporate card + spend management platform ของ US ที่มีลูกค้า 50,000+ ธุรกิจ) ปล่อย Ramp AI Index ฉบับเดือนพฤษภาคม วัดจาก actual spend pattern ของลูกค้า เผยตัวเลขที่ทำให้วงการ enterprise AI ใจหายตามกัน: **เดือน เม.ย. 2026 Anthropic ครอง 34.4% ของธุรกิจ US ที่ใช้ AI vendor / OpenAI ครอง 32.3%** — Anthropic แซง OpenAI เป็นครั้งแรกในประวัติศาสตร์ (Anthropic +3.8pp เดือนเดียว, OpenAI −2.9pp)

เส้นโค้งระยะยาวยิ่งหนัก — Anthropic share ของธุรกิจ ขยับจาก 0.03% มิ.ย. 2023 → 7.94% เม.ย. 2025 → **34.4% เม.ย. 2026** = โต **4 เท่าในปีเดียว** ขณะ OpenAI ในช่วง 12 เดือนเดียวกันโตแค่ 0.3 percentage point จาก ~32% → 32.3% — ไม่ใช่ Anthropic แค่ขึ้นมา แต่ OpenAI ก็ stagnate ในมิติ enterprise; Menlo Ventures (VC ที่ทำ LLM enterprise spend report ใหญ่ที่สุดของวงการ) ก็ออกตัวเลขสนับสนุน: enterprise LLM spend ปี 2026 แตะ **$8.4B** และ Anthropic เป็น share leader

Driver หลักคือ **Claude Code** — Anthropic ยืนยันใน earnings call ที่ผ่านมาว่า Claude Code เป็น product โตเร็วที่สุดในประวัติบริษัท; Pragmatic Engineer ของ Gergely Orosz ทำ survey ใน developer community พบว่า ~70% ของ engineering team Fortune 500 ที่ใช้ AI coding agent ตอนนี้เป็น Claude Code (เทียบ Cursor + Codex รวมกัน ~25%); Anthropic ออกประกาศ 13 พ.ค. ขยาย Claude Code weekly limit อีก 50% ถึง 13 ก.ค. สำหรับ Pro/Max/Team/Enterprise — เป็นการปรับ capacity ครั้งที่ 3 ใน 5 สัปดาห์

วันต่อมา 14 พ.ค. Anthropic เปิด **Claude for Small Business** — product package ที่ embed Claude เข้า business software ที่ SMB ใช้อยู่แล้ว (QuickBooks, HubSpot, Shopify, Google Workspace) พร้อม workshop tour ฟรี 10 เมืองทั่ว US รับ 100 entrepreneur/เมือง — เป็นการลงไปจับ SMB segment ที่ OpenAI ทิ้งเปล่ามาตลอด; เกือบพร้อมกัน OpenAI ตอบโต้: คืน support ให้ outside agent tool บน Claude Code แต่ใส่ "credit meter แยก" — แปลว่าลูกค้า paid Claude plan ที่ใช้ tool ภายนอกจะถูก meter เพิ่ม (ตีความได้ว่า Anthropic ปกป้อง margin หรือบีบให้ใช้ tool ของ Anthropic เอง)

## ทำไมสำคัญ

Inflection point นี้สำคัญไม่ใช่เพราะ "Anthropic ชนะแล้ว" — Ramp เองออกตัวว่า "1-month lead ไม่ใช่ moat" — แต่สำคัญเพราะมัน falsify สมมติฐาน 2 ข้อที่วงการเชื่อมานาน: (1) **"OpenAI brand recognition = enterprise default"** — ผิด consumer/SMB ยังเรียก ChatGPT แต่ enterprise spending vote ด้วย Claude (2) **"Multi-model routing คือทางออก"** — ผิด ในความเป็นจริง enterprise CIO ส่วนใหญ่เลือก **single primary model** แล้วเสริมด้วย secondary 1 ตัว ไม่ใช่ build router ที่ pick best model ทุก query

อีกข้อที่อ่านออกคือ **Anthropic ชนะที่ "product" ไม่ใช่ "model"** — Claude Sonnet 4.6 และ Opus 4.7 เก่งกว่า GPT-5 บางมิติแต่ไม่ได้ทิ้งห่าง; สิ่งที่ทำให้ Anthropic โตคือ Claude Code (product ที่ developer ลงทุนเอง bring-your-own-team), Computer Use (vision-based action), Claude Security (vertical product) — ทั้งหมดเป็น product layer บน model ที่ OpenAI ยังไม่ได้สร้าง equivalent; OpenAI ผูกตัวเองอยู่กับ ChatGPT consumer + Codex (which ตามมาทีหลัง)

มุมที่ลึกกว่าคือ **Anthropic เริ่มเปลี่ยน pricing model เป็น metering** — credit meter ใหม่บน outside agent tool คือ signal ว่า Anthropic เริ่มลด "all-you-can-eat" subscription แล้วบีบเก็บ usage-based; pattern ตรงนี้บอกว่า Anthropic ทราบดีว่า heavy user (เช่น dev ที่รัน Claude Code 24/7 + tool external) กิน margin ลง — และเลือก protect margin ก่อนสร้าง share เพิ่ม; OpenAI เลือกตรงกันข้าม — ขยาย Codex limit + ลดราคา agent token เพื่อดูดผู้ใช้กลับ; การที่ 2 บริษัทเลือก strategy ตรงข้ามใน inflection ที่ลึกแบบนี้บอกว่า **ตลาดยังไม่ converge** — ลูกค้าจะมีอำนาจต่อรองสูงในอีก 6–12 เดือน

## มุม OpenBridge

ตรงประเด็น — **buyer mid-market/SMB ที่ Anthropic เพิ่งลงจับ คือ segment เดียวกับที่ OpenBridge ตั้งเป้า**; ดี/เสียอย่างไร?

ดี: Anthropic ใช้ Claude for Small Business เปิดตลาด SMB ที่ก่อนหน้านี้ขาดความเชื่อมั่นว่า AI ใช้ได้จริง — เมื่อ SMB เห็น Claude embed ใน QuickBooks/Shopify จะมาถามต่อว่า "เชื่อม Claude เข้า ERP/CRM ของฉันได้ไหม" — OpenBridge เป็นคำตอบ; pattern เดียวกับ Twilio ที่โตได้เพราะ AWS เปิดตลาด API ให้ตลาดยอมรับก่อน

เสีย: ถ้า Anthropic + Shopify/QuickBooks ทำ deep integration ที่ครอบคลุม use case ทั่วไปของ SMB (เช่น invoice automation, customer support, marketing copy) ลูกค้า SMB จะไม่ต้องการ integration platform ตัวกลาง — Anthropic Direct-to-SMB เป็น threat ที่ต้อง watch; OpenBridge ต้อง position ที่ **"vertical workflow ที่ Anthropic global ไม่ลงมา"** — ภาษาไทย, ระบบบัญชี Express/AccountStar/CRM ของ SEA, integration กับ LINE/Lazada/Shopee — ที่ Anthropic ไม่มี incentive ทำเอง

อีก insight: **enterprise CIO 2026 เลือก primary model เป็น Claude มากกว่า GPT** = ทุก integration platform ใน 12 เดือนข้างหน้าต้อง support Claude เป็น first-class citizen ไม่ใช่ option; OpenBridge ควรชู "Claude-native" + "MCP-first" เป็น default narrative — ตามที่ Anthropic เพิ่ง donate MCP ให้ Linux Foundation เมื่อ ธ.ค. 2025
