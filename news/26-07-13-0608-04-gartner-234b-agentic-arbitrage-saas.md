# Gartner ปักธง "agentic arbitrage" — $234B ของ enterprise SaaS spend เสี่ยงถูก agent กินภายในปี 2030 (~20% ของตลาด)

## TL;DR
- 1 กรกฎาคม 2026 **Gartner** ปล่อย forecast: **$234 พันล้าน** ของ enterprise application software spend "at risk" จาก **agentic arbitrage** ภายในปี 2030 — ประมาณ **20% ของ enterprise SaaS spending** ทั้งหมด
- Definition: agent complete task ข้ามหลาย system โดยตรง → **user ไม่ต้องเข้า UI ของ SaaS แต่ละตัว** → link ระหว่าง user growth กับ revenue growth **หลุด** สำหรับ vendor UI-heavy
- Gartner ชี้ว่า **AI-native startup + service provider** ที่ position เป็น "agentic layer" จะกินทั้ง existing spend + budget ใหม่ที่ ROI ปลดล็อก — ส่วน incumbent SaaS ที่คิด pricing ตาม seat/UI usage จะกระทบตรง

## เกิดอะไรขึ้น
Gartner ไม่ใช่ vendor แต่เป็น analyst house ที่ CIO ใช้เป็น proxy ในการเซ็นงบ — เวลา Gartner ปักธงตัวเลข **$234 พันล้านจาก enterprise application software spend "at risk" จาก agentic arbitrage ระหว่างวันนี้ถึงปี 2030** จึงมี weight คนละแบบกับ VC blog. บทวิเคราะห์ (1 ก.ค. 2026) กำหนดคำใหม่ **agentic arbitrage**: ปรากฏการณ์ที่ agent complete task ข้ามหลาย system ให้เสร็จโดยที่ user ไม่ต้องเข้าไปใช้ SaaS UI แต่ละตัวเอง

George Brocklehurst, Managing VP ที่ Gartner ใช้คำว่า **"agentic AI changes the economics of software"** — เพราะ agentic system deliver outcome โดยตรง, bypass user-experience layer, ทำให้ software "invisible". สิ่งที่กระทบไม่ใช่แค่ margin แต่คือ **link ระหว่าง user growth กับ revenue growth**: SaaS pricing ที่คิดต่อ seat, ต่อ session, ต่อ active user จะ decouple จาก value ที่ส่งมอบ. Gartner บอก 20% ของ SaaS spend ทั้งหมดจะไหลผ่าน agent layer ภายในปี 2030 — ไม่ใช่ replacement ทันที แต่คือ **redirection ของ dollar flow**

Winner ตาม Gartner คือ **AI-native startup + service provider** ที่ทำตัวเป็น agentic layer across enterprise systems — deliver outcome ไม่ใช่ feature, และ redesign workflow รอบ ๆ AI ให้ลูกค้า. คนกลุ่มนี้ **ไม่ได้แค่กิน spend เดิมของ SaaS incumbent — แต่จะกิน incremental budget ที่ ROI ใหม่ปลดล็อก** ซึ่งอาจ 2-3x เท่าของ SaaS budget เดิมในบาง vertical

## ทำไมสำคัญ
report นี้ครบ trifecta ที่ทำให้ CIO ต้องประชุมด่วน: **(1) ตัวเลข analyst brand-name เอามาใส่ board deck ได้**, **(2) คำที่ติดหู "agentic arbitrage" จะกลายเป็น shorthand ใน quarterly review**, **(3) timing ตรงกับ budget planning ครึ่งหลังของปี**. ผลกระทบระยะสั้นคือ **procurement team จะเริ่มถาม vendor ทุก SaaS ว่า "agent story ของคุณคืออะไร"** ก่อน renewal — Salesforce, ServiceNow, Workday, SAP ที่ชี้ Agentforce, Now Assist, Illuminate, Joule จะได้ผ่าน filter ง่ายกว่า

ที่น่าคิดคือ **contrast** กับ Stanford AI Index ที่ Enabridge เคยรายงาน — 89% ของ agent deployment fail ใน production, Monte Carlo บอก 64% ของ enterprise ที่ rush agent deployment เห็น cost มากกว่า benefit. Gartner report ไม่ได้ปฏิเสธเรื่องนี้ — บอกกลาย ๆ ว่า **enterprise ที่ shift ทันจะเป็นผู้ชนะ, enterprise ที่ shift ช้าจะโดน redirect budget โดย vendor ที่ทำเป็น**. ในความเป็นจริง 2 report ตรงกัน — technology work, execution risk สูง, และ window การเลือกข้างเปิดอีกไม่นาน

## มุม AI Agent Platform
สำหรับ **builders**: pricing model ที่ทำต่อ seat ต่อ session ต่อ token จะต้องย้ายไป **outcome-based / value-based pricing** — คิดต่อ ticket resolved, ต่อ document processed, ต่อ deal closed. builder ที่ยังคิด pricing แบบ "$X ต่อ user ต่อเดือน" จะ lose ตัวเปรียบเทียบตอน customer ทำ ROI review เพราะ agent user ไม่มีอยู่จริง. เตรียม **instrumentation ที่วัด outcome ได้จริง** — เป็น moat มากกว่า model quality. สำหรับ **users/business**: ใช้ report นี้เป็น mandate ใน budget cycle ครึ่งหลัง — audit SaaS ทุกตัวว่า "agentic layer" ของ vendor คืออะไร มี MCP endpoint ไหม, ให้ agent 3rd-party call ได้ไหม, ราคาต่อ outcome เท่าไร. vendor ที่ตอบไม่ได้ ตัด renewal หรืออย่างน้อยลดขนาด commitment ให้เหลือ 12 เดือน. สำหรับ **ecosystem**: **service provider / SI จะได้ role ใหญ่ขึ้น** — Accenture, Deloitte, EY, TCS, Infosys ที่ position เป็น "who reweaves workflow around agent" จะเป็นตัวกลางที่ redirect budget flow จริง ๆ. ส่วน SaaS incumbent ต้องเลือก: (a) เปิด API + MCP + agent SDK ให้กลายเป็น "system of record ที่ agent เชื่อมได้", หรือ (b) สร้าง agent layer เอง แข่งกับ AI-native. คนที่ทำทั้งสองแบบ เช่น Salesforce, Microsoft, SAP มีโอกาสรอด — คนที่ทำแค่ back-end system ไม่มี agent story จะเจอราคาถูก compress ใน 3-4 ปีข้างหน้า
