# Daily Brief — 26-05-07 รอบ 06:12 (Bangkok)

## Theme ของรอบ

**5–6 พฤษภาคม คือวันที่ตลาด agent เปลี่ยน phase อย่างชัดเจน — phase 1 (2024–25) แข่งสร้าง agent, phase 2 เริ่มแล้ว: แข่งสร้าง "ระบบควบคุม agent"** ServiceNow ที่ Knowledge 2026 เปิด AI Control Tower ที่มาพร้อม **real-time kill switch** + 30 connectors ครอบ AWS/GCP/Azure + SAP/Oracle/Workday — Bill McDermott พูด: "Enterprises need AI that senses, decides, and securely acts" Fortune พาดหัว "Your company's AI could delete everything in 9 seconds" สะท้อนความกังวลที่ CIO/CISO ทุกที่กำลังเจอ

วันเดียวกัน Collibra ออก AI Command Center, Nvidia ขยายความร่วมมือ governance, Accenture เปิด Forward Deployed Engineering program — pattern ชัดว่าตลาด pile-on "control plane" รอบใหม่ ในขณะที่ฝั่ง protocol — **CopilotKit ปิด $27M (Glilot/NFX/SignalFire)** และเปิดเผยว่า **AG-UI** ที่บริษัทเป็นเจ้าภาพถูก adopt โดย Google, Microsoft, Amazon, Oracle เป็น standard กลาง — ครบ 3 protocol layer ของ agent stack: MCP (data), A2A (agent-to-agent), AG-UI (UI)

ฝั่ง vertical agent มีสองดีลใหญ่: **Nova Intelligence ปิด Series A $31.5M (Chemistry lead)** บน $89B SAP S/4HANA migration wave — case study Kyndryl อ้างลด manual effort 75%, cost 50%; **Blitzy ปิด $200M @ $1.4B (Northzone lead)** สำหรับ autonomous coding agent — strategic round มี Liberty Mutual + Erie Insurance ลงพร้อมกัน ทั้งสองคืออาการของ pattern "vertical agent ที่เกาะ regulatory mandate / legacy debt" — ตลาดที่ลูกค้าไม่มี option ไม่ทำ

Pattern หลักที่ run ผ่านทั้งสี่เรื่อง: **agentic landscape กำลัง vertical-ize เร็วกว่า horizontal-ize** — control plane (ServiceNow), UI protocol (CopilotKit/AG-UI), SAP migration (Nova), legacy code modernization (Blitzy) ล้วนเป็น layer ที่ specialized — และทุกชั้นล้วนเปิด opportunity adjacent ให้ integration platform แบบ OpenBridge

## TL;DR สำหรับ Yoh

- **Phase 2 ของ agent market เริ่มแล้ว** — phase 1 แข่งสร้าง agent, phase 2 แข่งสร้าง control plane; ServiceNow ใช้ leverage Fortune 500 80% installed base วาง kill switch ทับทุก agent runtime; pure-play governance startup (Lasso/CalypsoAI/Credo) ปีน acquisition curve ยาก; gap ระหว่าง deploy (91%) vs governance (48%) คือ market opportunity 43-point
- **3-layer protocol ของ agent ครบแล้ว** — MCP (Anthropic) สำหรับ data/tools, A2A (Google) สำหรับ agent คุย agent, AG-UI (CopilotKit) สำหรับ agent คุย UI; CopilotKit ดึง Google/Microsoft/Amazon/Oracle ลงเรือพร้อมกันเป็น signal protocol convergence; ใครยึด protocol ชั้นไหน เก็บภาษีชั้นนั้น
- **Vertical agent ที่ระดมทุนได้แพง = ตัวที่เกาะ mandate ตลาดที่ขยับไม่ได้** — Nova เกาะ SAP S/4HANA deadline 2030 ($89B), Blitzy เกาะ legacy COBOL/Java ของ insurer; ทั้งสองตลาดลูกค้าไม่มี option ไม่ทำ — pattern นี้ใช้ใน TFRS, e-Tax, e-Withholding ของไทยได้
- Action ตรงสำหรับ OpenBridge: (1) **ออกแบบ audit log + revocation API** ที่ honor signal kill switch จาก ServiceNow/Collibra ตั้งแต่วันนี้ — เป็น "preferred partner" ของ control plane (2) **ประเมินทำ AG-UI SDK** — ถ้า dev ที่ใช้ CopilotKit ต่อ OpenBridge ได้ใน 5 นาที จะกลายเป็น default integration layer ของ Fortune 500 (3) **redesign rate-limit จาก per-workflow เป็น per-agent quota + connection pool** — agent จะมาในระดับ 1000x ไม่ใช่ 10x (4) **เปลี่ยนภาษา GTM จาก feature-based เป็น outcome-based** — copy lesson Nova "75% เร็วกว่า, 50% ถูกกว่า" แทน "N connectors"
