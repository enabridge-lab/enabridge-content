# Dust ระดม $40M — Paris สตาร์ตอัพ "multiplayer AI" ที่ดีพลอย 300,000 agent + zero churn 2025

## TL;DR
- 18 พ.ค. Dust (Paris, ก่อตั้ง 2023 โดย Gabriel Hubert + Stanislas Polu จากอดีต Stripe/OpenAI) ปิด Series B **$40M** นำโดย **Sequoia + Abstract** พร้อม Snowflake Ventures + Datadog — รวมระดมแล้ว > **$60M**
- ตัวเลข production: **3,000 องค์กร**, **51,000 MAU**, **300,000 agent** ที่ deploy บน platform, **zero churn ตลอดปี 2025**
- Thesis "**multiplayer AI**" — แทนที่ assistant แบบ single-player (ChatGPT 1-1) ให้ทีม + agent ทำงานบน workspace เดียว, shared context, bidirectional notification — เป็น direct challenger ของ Glean + Copilot Studio

## เกิดอะไรขึ้น

18 พฤษภาคม 2026 Dust — สตาร์ตอัพ enterprise AI agent ที่ Gabriel Hubert (อดีต Stripe Paris) กับ Stanislas Polu (อดีต OpenAI researcher) ตั้งขึ้นเมื่อ 2023 — ประกาศปิด Series B **$40M** นำโดย **Sequoia Capital** + **Abstract Ventures** พร้อม **Snowflake Ventures** + **Datadog**. รวมระดมแล้วเกิน $60M. ราวด์นี้ทำให้ Dust กลายเป็น Paris-based agentic AI champion ที่ได้ Tier-1 SV venture หลัง — Sequoia ไม่ลงตัวเล็กง่าย ๆ

ตัวเลข production ที่ Dust เปิดเผย — **3,000 องค์กร** ใช้ platform, **51,000 monthly active user**, **300,000 agent** ที่ deploy + run อยู่บน Dust workspace, และ key signal คือ **zero customer churn ตลอดปี 2025**. เป็นเลขที่หายากใน B2B SaaS (industry benchmark ~5-10% gross churn/year) — แปลว่า platform "stick" จริง + retention ดีกว่า Glean, Copilot Studio, ChatGPT Enterprise

Thesis ของ Dust ที่ทำให้ Sequoia ลงคือ **"multiplayer AI"** — observation ว่า enterprise AI ส่วนใหญ่ตอนนี้เป็น single-player (1 dev คุยกับ ChatGPT, 1 sales rep คุย Copilot) ซึ่งไม่ scale เพราะ knowledge + decision ไม่ shared. Dust สร้าง workspace ที่ทีม + agent ทำงานบน project เดียวกัน — shared context, shared notification, bidirectional access. Agent ที่ทีม sale ใช้ qualifying lead วันนี้ เห็น context ของทีม CS ที่ทีม ops หา revenue churn เมื่อวานได้ทันที — เพราะอยู่บน workspace เดียว ไม่ใช่ ChatGPT thread แยก

Stanislas Polu บอก SiliconANGLE: "เราจะ push 3 frontier พร้อมกัน — (1) agent ที่ learn + improve อัตโนมัติเมื่อใช้, (2) collaboration primitive ที่ human กับ agent เป็น equal co-contributor (ไม่ใช่ agent เป็น tool ของ human เฉย ๆ), (3) infrastructure ที่ทำ governance + orchestration predictable ที่ enterprise scale". เงิน $40M จะลงไป engineering ทั้ง 3 ขา + ขยาย US sales (HQ Paris แต่ growth ส่วนใหญ่จาก US enterprise)

## ทำไมสำคัญ

Dust เป็น **proof point ที่ undercovered ของ EU agentic AI champion** ที่ส่งมือถึงระดับโลก. ปีที่แล้ว market narrative คือ "EU แพ้ AI race เพราะ regulator + ทุนน้อย" — แต่ Mistral (model), Owkin (healthcare), Helsing (defense), Dust (agent) กำลังโต silent. Dust จัดอยู่ใน band เดียวกับ Sierra (US, $15.8B) + Glean ($7B) ในด้าน thesis แต่ valuation ต่ำกว่า 10x — ถ้า Sequoia push retention story ของ Dust ออก market ครบ ปีหน้า Dust อาจ raise Series C ที่ $1-2B valuation

มอง 12-18 เดือนข้างหน้า — **"multiplayer" vs "single-player" คือ axis ใหม่ที่ enterprise AI vendor แข่งกัน**. Glean สร้าง enterprise search ที่ทุกคนใช้แต่ละคน ChatGPT Enterprise เก็บ context per user, Microsoft Copilot push Teams integration เพื่อ multiplayer แต่ adoption ยัง 1-1, Sierra ทำ customer-facing agent (B2C) ไม่ใช่ internal team. Dust ที่ build จากวันแรกเป็น team-first + agent-as-coworker คือ position ที่ unique — ที่ Microsoft ที่ใหญ่ที่สุดต้อง re-architect แพ้ Dust ที่เริ่มใหม่

อีก signal สำคัญ — **300,000 agent deployed บน 3,000 องค์กร = 100 agent/org เฉลี่ย**. ตัวเลขนี้บอกว่าใน mid-market firm หนึ่ง — ไม่ใช่ "1 agent ทำได้ทุกอย่าง" แต่ "100 agent specialized" ที่ทำงานเฉพาะ (sales agent, billing agent, onboarding agent, recruiting agent, IT helpdesk agent ...). Pattern นี้ตรงกับสิ่งที่ Notion ทำเมื่อ 5 ปีก่อนกับ workspace + template — แต่ scale ที่ Dust ทำได้ใน 18 เดือน suggesting **agent proliferation curve เร็วกว่า template/app proliferation 5-10x**

## มุม OpenBridge

ตรงกระทบ. Dust = direct competitor ในระยะกลาง — แต่ Dust ขาย workspace platform, OpenBridge ขาย integration layer ใต้ platform. **opportunity เร็วที่สุดคือ "official Dust connector partner สำหรับ SEA stack"** — ติดต่อ Dust BD ภายใน 30 วัน เสนอ build connector kit สำหรับ KBank API + LINE Notify + LINE OA + ShopeePay + GrabBusiness + AIS messaging + PromptPay + ฯลฯ — ที่ Dust ตอนนี้ไม่มี local team ทำเอง. ถ้า Dust adopt OpenBridge connector → revenue stream ใหม่ + reverse referral ของลูกค้า EU/US ที่ expand เข้า SEA

มอง defensive — **multiplayer AI thesis คือสิ่งที่ OpenBridge ต้อง absorb เป็น product principle**. Integration layer ที่ build ในยุค single-player AI (Zapier ออกแบบ trigger/action 1-1) จะ obsolete ในยุค team + agent collaboration. OpenBridge ต้องออกแบบ — (1) **shared context primitive** — เก็บ history ของ workflow execution ให้ทีม + agent ทุกคนเห็นได้, (2) **bidirectional notification** — agent เห็น state ของ workflow ก่อนเลือก action, (3) **policy ระดับ team ไม่ใช่ per-user** — ACL ที่ map กับ team structure ของ org. นี่คือ feature ที่ OpenBridge ควร roadmap ก่อนปลายปี ก่อนที่ Dust หรือ Glean จะกินตลาดทั้ง horizontal

อีก insight ใหญ่ — **zero churn = สัญญาณว่า lock-in เร็วกว่าที่คิด**. ลูกค้า enterprise ที่ deploy 100 agent บน Dust ใน 18 เดือน ไม่ย้ายเพราะ workflow + context + audit log + permission rewire ใหม่หมด. ระยะ 12-18 เดือนหน้า — ลูกค้า SEA ที่กำลังตัดสินใจ "ใช้ platform ไหน build agent ใน org" คือ buying decision ที่ stick ไป 5-7 ปี. **OpenBridge ต้อง ship "agent workspace" หรือ "embedded integration layer ใต้ Dust/Sierra/Glean" ก่อนปลายปี — ไม่งั้นเสีย window**. ถ้าเลือก path 2 (embed) — ต้องเร่งทำ partnership + reference architecture ที่ Dust + Glean ยอม certify
